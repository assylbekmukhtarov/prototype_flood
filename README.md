# Прототип: мониторинг снега и паводков

Работает без GEE и без токенов. Данные Sentinel-2 скачиваются напрямую из AWS через STAC-каталог Element84.

## Как пользоваться

1. Нарисовать прямоугольник или полигон на карте инструментом рисования
2. Задать даты и пороги в боковой панели
3. Температура подтягивается автоматически из Open-Meteo по дате «после»
4. Нажать **«Анализировать»** — ждать 5–60 секунд
5. На карте появятся три слоя: снег (синий), вода (голубой), новое затопление (красный) (небольшой баг может быть, что пиксели сдвинуты)
6. Слои можно включать/выключать чекбоксами в легенде справа снизу
7. Кнопка **«Показать RGB-снимок»** — накладывает реальный снимок Sentinel-2 поверх карты (небольшой баг может быть, что пиксели сдвинуты)
8. Кнопка **«Скачать CSV»** — экспортирует метрики

## Лицензии и ограничения на коммерческое использование

| Сервис / данные | Лицензия | Коммерческое использование |
|---|---|---|
| **Open-Meteo** (температура) | CC BY 4.0 | Бесплатно до 10 000 запросов/день. Свыше — нужна коммерческая подписка (~$30/мес). Обязательна атрибуция. |
| **Sentinel-2** (снимки ESA) | Copernicus Data Policy | Свободно для любых целей включая коммерческие. Обязательна атрибуция ESA / Copernicus. |
| **Element84 earth-search** (STAC API) | Публичный, бесплатный | Регистрация не нужна. Нет явных лимитов, но не рекомендуется для высоких нагрузок без договорённости с провайдером. |
| **AWS S3** (COG-файлы Sentinel-2) | AWS Open Data Program | Регистрация не нужна. Данные оплачены ESA в рамках AWS Open Data — чтение бесплатно для всех. Платным становится только исходящий трафик если развернуть собственный сервер на AWS вне региона us-west-2. |
| **OpenStreetMap** (тайлы карты) | ODbL | Данные OSM коммерчески свободны при атрибуции. Публичный тайловый сервер `tile.openstreetmap.org` — только для прототипов и разработки. Для production нужен платный провайдер (Maptiler, Mapbox и др.) или собственный тайловый сервер. |
| **Leaflet** | BSD 2-Clause | Полностью свободна для коммерческого использования. |
| **FastAPI / uvicorn** | MIT | Полностью свободна для коммерческого использования. |
| **rasterio / shapely / pyproj** | BSD / MIT | Полностью свободны для коммерческого использования. |

**Вывод для production:** основные риски — Open-Meteo при высокой нагрузке и OSM тайловый сервер. Остальное коммерчески свободно и не требует регистрации.

## Структура

```
prototype_flood/
├── main.py          # FastAPI: /api/analyze, /api/export-csv, /api/rgb, /api/temperature
├── sentinel.py      # поиск снимков через STAC, COG-чтение по bbox, исторический профиль
├── analysis.py      # NDSI, MNDWI, маски, площади, весовая модель риска, GeoJSON
├── requirements.txt
├── Dockerfile
├── docker-compose.yml
└── static/
    └── index.html   # весь фронт: Leaflet карта + форма + результаты + легенда с чекбоксами
```

## Запуск через Docker

```bash
cd prototype_flood

# собрать образ и запустить
docker compose up --build -d

# или вручную
docker build -t flood-monitor .
docker run -p 8000:8000 flood-monitor
```

Открыть в браузере: http://localhost:8000

Остановить: `docker compose down`

```bash
cd prototype_flood

# создать виртуальное окружение

pip install -r requirements.txt
```

## Эндпоинты API

| Метод | URL | Описание |
|---|---|---|
| GET | `/` | Главная страница |
| POST | `/api/analyze` | Основной анализ: маски + метрики + GeoJSON + риск |
| POST | `/api/export-csv` | Скачать CSV с метриками |
| GET | `/api/rgb` | PNG RGB-снимок (B4/B3/B2) для наложения на карту |
| GET | `/api/temperature` | Средняя температура за период из Open-Meteo (ERA5 или прогноз) |

## Параметры запроса /api/analyze

```json
{
  "bbox": [minLon, minLat, maxLon, maxLat],
  "date_snow_start": "2025-02-01",
  "date_snow_end": "2025-03-15",
  "date_before": "2024-09-01",
  "date_after": "2025-04-01",
  "snow_threshold": 0.4,
  "water_threshold": 0.0,
  "melt_rate_threshold": 50.0,
  "temp_forecast": null,
  "historical_years": 10
}
```

## Какие снимки скачиваются

За каждый запрос скачивается до четырёх снимков:

| Снимок | Период | Каналы | Назначение |
|---|---|---|---|
| Снежный | `date_snow_start` — `date_snow_end` | B3, B11 | NDSI, маска снега |
| До паводка | `date_before` ± 7 дней | B3, B11 | MNDWI, маска воды до |
| После паводка | `date_after` ± 7 дней | B3, B11 | MNDWI, маска воды после |
| RGB (по запросу) | `date_after` — `date_after` + 30 дней | B4, B3, B2 | Визуальная подложка |

Для каждого периода выбирается снимок с минимальной облачностью (< 20%). Если снимка нет — возвращается ошибка 404.

## Оценка риска

Весовая модель из четырёх факторов (итоговый score 0–100):

| Фактор | Вес |
|---|---|
| Снег vs историческое среднее | 40% |
| Скорость таяния vs порог | 25% |
| Прогноз температуры | 20% |
| Текущее затопление vs исторический максимум | 15% |

Градации: `low` < 30, `medium` 30–70, `high` > 70.

## Ограничения прототипа

- Время ответа: 5–60 сек в зависимости от размера bbox, скорости AWS и числа исторических лет
- Если за период нет снимков с облачностью < 20% — вернёт ошибку 404
- Рекомендуемый размер bbox: не более 1° × 1°
- Экспорт GeoTIFF не реализован — маски отдаются как GeoJSON
